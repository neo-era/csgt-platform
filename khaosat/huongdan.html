1 Giới thiệu

Lighting Survey là ứng dụng web bản đồ dùng để khảo sát và quản lý hệ thống chiếu sáng công cộng. Hoạt động trực tiếp trên trình duyệt, không cần cài đặt, có thể dùng offline sau lần tải đầu tiên, và cài như app native (PWA) trên điện thoại lẫn máy tính.

ℹ️
Dữ liệu đồng bộ hai chiều: ghi realtime vào Google Sheets qua Apps Script và lưu snapshot Excel trên GitHub. Hỗ trợ đa địa bàn (14 quận/huyện/xã) với sheet riêng cho từng nơi.

🚀 Tính năng nổi bật

🗺 Bản đồ & Marker

  • 4 lớp bản đồ nền: Google Maps (default), OpenStreetMap, Vệ tinh Google, Vệ tinh + tên đường
  • 6 loại đối tượng: Trụ STK / Trụ trang trí / Trụ HTLT / Trụ TTLT / Tủ chiếu sáng nổi / Tủ chiếu sáng ngầm
  • Icon động: hình dáng đổi theo số lượng bóng đèn (1-4 bóng), badge công suất + loại đèn (vd 2L100 = 2 bóng LED 100W)
  • Cluster thông minh: gộp marker khi zoom < 15, hiện riêng lẻ khi zoom ≥ 15, hiện nhãn tên khi zoom ≥ 17
  • Xoay bản đồ: la bàn tương tác + phím tắt [ / ] ± 5°, xoay 2 ngón touch
  • Widget tỷ lệ: chọn nhanh 1:200 → 1:25000 (chuẩn A3) để map zoom đến mức tương ứng

📡 Định vị GPS — hỗ trợ RTK Tersus Luka

  • 2 chế độ: Phone GPS (±3-10m, mặc định) hoặc RTK qua mock location (±1-2cm khi FIXED)
  • Tracking bar realtime: accuracy + sample counter + auto-accept countdown 3s
  • Status badge RTK: 🟢 FIXED / 🟡 FLOAT / 🔵 DGPS / ⚪ SPS
  • Multi-sample averaging (MAD outlier filter) khi dùng phone GPS — sai số giảm ~√8 ≈ 2.83 lần
  • Precision tự động: phone lưu tọa độ 6 chữ số, RTK lưu 8 chữ số (mm-level)
  • Auto-detect: cảnh báo khi chọn RTK nhưng chưa setup mock location

📝 Quản lý đối tượng — 25 cột data

  • Tên trụ, tủ điều khiển, loại trụ/cần/đèn, công suất, mã PE, đường, phường/xã
  • Ảnh chụp tự động upload GitHub, hỗ trợ lazy load
  • Marker gốc (cùng basename / cùng tủ), khoảng cách cáp (auto tính Haversine)
  • Loại cáp (nổi/ngầm) — vẽ với màu & kiểu nét khác nhau
  • Reverse geocoding tự động điền Đường + Phường/Xã (OpenStreetMap)
  • VN2000-X/Y tự tính từ Lat/Lon (Gauss-Krüger, ellipsoid GRS80)
  • Độ chính xác + chế độ GPS lưu để audit về sau
  • Cho phép kéo marker để chỉnh vị trí (toggle "Bật chỉnh vị trí")

🔌 Sơ đồ cáp

  • Vẽ tự động đường cáp từ trường Marker gốc + nhãn khoảng cách
  • 2 loại cáp: nổi (xanh dash 10 4) và ngầm (đỏ dash 2 3 8 3)
  • Multi-select tủ điều khiển để vẽ riêng tuyến
  • Chế độ sửa cáp trực tiếp trên bản đồ (xóa đoạn / đổi điểm gốc)
  • Phát hiện vòng lặp cáp tự động — hiện chain trụ bị lỗi

🖨 In bản vẽ kỹ thuật PDF

  • Xuất PDF A3 hoặc A4 ngang theo chuẩn bản vẽ kỹ thuật (tiêu đề, bảng ký hiệu, khung tên)
  • Tỷ lệ chính xác (1:500 → 1:25000) — đo bằng thước trên PDF khớp tỷ lệ chọn
  • Fractional zoom + math chuẩn (H_screen + mmH)
  • Bảng ký hiệu 3 cột (Ký hiệu | Số lượng | Đơn vị) — icon kích thước thật, đếm chính xác
  • Auto-fit chiều cao footer theo số loại marker, fallback compact khi nhiều loại
  • Auto-fill tên bản vẽ từ filter tủ điều khiển
  • Xem trước trước khi xuất, xoay map tùy chỉnh
  • Persist toàn bộ field form qua localStorage (trừ ngày)

🔍 Tìm kiếm & Lọc

  • Tìm trụ: theo tên / ID / tọa độ lat,lon — match không dấu, không phân biệt hoa thường, bỏ qua khoảng trắng & ký tự đặc biệt (vd gõ duong tran hung dao match Đường Trần Hưng Đạo)
  • Filter overlay: lọc theo Loại marker, Người KS, Phường/Xã, Đường, Tủ điều khiển (multi-select có search)
  • Bộ lọc apply tự động vào Sơ đồ cáp, In bản vẽ, Xuất báo cáo

🌍 Đa địa bàn — 14 sheet

  • TP HCM (10): Q1, Q3, Q5, Q8, Q10, Q11, Phú Nhuận, Bình Thạnh, Tân Bình, Tân Phú
  • Bình Dương (3): Xã Bàu Bàng, Xã Trừ Văn Thố, Phường Bến Cát
  • Long An (1): Cần Giuộc (file Google Sheet riêng — qua EXTERNAL_SPREADSHEET_IDS)
  • ID tự sinh theo prefix địa bàn (TQ/Q1/Q3/.../CG)
  • Phân vùng theo tài khoản (cột vung trong sheet TaiKhoan) — user chỉ thấy địa bàn được phép

☁ Đồng bộ & Export

  • Google Sheets realtime — mỗi thao tác (thêm/sửa/xóa/move) ghi ngay qua GAS Web App
  • Batch sync cho marker đã kéo — nút "Cập nhật" đẩy hàng loạt khi nhiều marker thay đổi vị trí
  • GitHub Excel — bấm 1 nút để snapshot toàn bộ data thành data/khaosat_<sheet>.xlsx, push lên repo
  • Version auto-bump mỗi commit push (GitHub Action) — hiển thị trên tab title
  • Excel báo cáo: 2 sheet (Chi tiết + Tổng hợp theo loại), lọc theo ngày/phường/loại
  • DXF CAD: tọa độ VN-2000 múi 3°, mở được AutoCAD / QGIS / BricsCAD
  • Log lịch sử: ghi mọi thao tác (delete/edit/move) vào sheet LichSu — chỉ admin xem được

👥 Phân quyền 4 vai trò

  • admin: đầy đủ + xem lịch sử thao tác
  • user: thêm/sửa/xóa + GitHub sync
  • user1: thêm/sửa, không được xóa
  • demo: chỉ xem bản đồ
  • Đổi tên đăng nhập / mật khẩu trực tiếp trong app (nút 🔑)

📱 PWA + Tối ưu mobile

  • Cài như app native trên Android, iOS, Windows, macOS
  • Service Worker cache offline (HTML network-first, map tiles + CDN libs cache-first ≤300 tile)
  • Icon mobile tối giản, shared icons khi không có badge (giảm DOM node ~80%)
  • Cluster aggressive hơn trên mobile, label chỉ hiện zoom ≥ 18
  • Tắt animation Leaflet trên mobile để mượt hơn
  • Lazy load ảnh trong popup, CSS media query ẩn nút ít dùng

⚡ Tối ưu hiệu năng

  • IndexedDB cache CSV (stale-while-revalidate) — load lần 2 < 0.3s
  • Chunked marker insertion + progress bar — không freeze UI khi load 1000+ marker
  • Lazy popup content — chỉ sinh HTML khi user click marker (tiết kiệm 1000+ lần gọi)
  • Zoom-tier progressive — zoom < 12 chỉ render tủ điều khiển (50-200 marker thay 14000+)
  • Lazy load CDN libs: ExcelJS, jsPDF, html2canvas — chỉ tải khi cần

🛣 Bonus tools

  • Chỉ đường xe máy tới marker (OSRM)
  • Mapillary Street View — xem ảnh đường phố tại vị trí marker
  • Lightbox ảnh: click ảnh marker để xem full
  • Reverse geocoding tiếng Việt (Nominatim OSM)

Yêu cầu hệ thống

  • Trình duyệt Chrome / Edge / Safari (phiên bản mới nhất) — Chrome khuyến nghị cho mobile
  • Kết nối Internet cho lần load đầu + sync data (sau đó dùng offline được phần lớn)
  • Tài khoản được cấp bởi quản trị viên (sheet TaiKhoan)
  • Phone Android có GPS + Location (nếu dùng tính năng định vị)
  • Tùy chọn: máy RTK Tersus Luka + Nuwa app + NTRIP correction (cho độ chính xác cm)

2 Đăng nhập & Phân quyền

Đăng nhập

1
Nhập tên đăng nhập và mật khẩu Do quản trị viên cấp. Tên đăng nhập không phân biệt chữ hoa/thường.
2
Nhấn "Đăng nhập" Máy chủ có thể mất 10–20 giây lần đầu (cold start). Nếu quá lâu, nút "Hủy" xuất hiện để thử lại.
3
Sau đăng nhập Bản đồ tải tự động. Tên người dùng hiển thị ở góc phải trên cùng. Nhấn vào thẻ tên để đăng xuất.

Đổi tên đăng nhập / mật khẩu

Nhấn ☰ → nhấn biểu tượng 🔑 bên cạnh tên người dùng → nhập mật khẩu hiện tại và thông tin mới → nhấn Lưu thay đổi.

⚠️
Sau khi đổi tên đăng nhập, phiên hiện tại sẽ kết thúc và yêu cầu đăng nhập lại bằng tên mới.

3 Giao diện bản đồ

Thanh trên (Top Bar)

  • ☰ — Mở bảng điều khiển (tài khoản, đồng bộ, GitHub, xuất dữ liệu...)
  • Ô tìm kiếm — Tìm theo tên trụ, ID hoặc tọa độ
  • Nhãn phiên bản dữ liệu — Hiện version file đang xem (VD: Dữ liệu v1.0.42)

Bảng điều khiển (☰)

  • Thẻ tài khoản — Tên user + vai trò; nhấn để đăng xuất. Nút 🔑 đổi thông tin đăng nhập.
  • Điều hướng — Dropdown "Chọn trang" chuyển giữa các địa bàn (quận/huyện/xã), lịch sử thao tác (admin), hướng dẫn. Các trang quản lý camera và đèn tắt/sự cố chỉ hiện với tài khoản admin.
  • Chỉnh vị trí & Đồng bộ — Bật/tắt kéo marker; nút Cập nhật đẩy vị trí lên Sheet.
  • Sơ đồ cáp — Hiển thị đường cáp nối các trụ; thêm nút Sửa cáp và Vẽ cáp.
  • In bản vẽ — Xuất PDF bản vẽ kỹ thuật theo bộ lọc hiện tại.
  • GitHub / Xuất dữ liệu — Lưu Excel, đồng bộ GitHub, xuất báo cáo, xuất DXF.

Chuyển địa bàn

Dropdown "Chọn trang" trong bảng điều khiển liệt kê tất cả địa bàn được phép. Chọn một địa bàn → bản đồ xóa marker cũ và tải dữ liệu mới của địa bàn đó.

💾
Dữ liệu mỗi địa bàn được cache trên trình duyệt (IndexedDB). Lần tải thứ hai gần như tức thì; nền app tự kiểm tra cập nhật mới.

Thanh dưới (Bottom Bar)

  • Vị trí — Căn giữa bản đồ về GPS hiện tại
  • + Thêm marker — Bắt đầu thêm trụ/tủ mới
  • Lọc — Mở bộ lọc marker trên bản đồ
  • Lưu — Tải xuống file Excel

Bản đồ

  • Lớp nền mặc định: Google Maps. Nút chuyển lớp (giữa bên phải) cho phép chuyển sang OpenStreetMap, Vệ tinh...
  • Zoom < 15: marker gộp cluster; Zoom ≥ 15: icon riêng lẻ; Zoom ≥ 17: nhãn tên trụ

Cụm điều khiển bản đồ (giữa bên phải)

  • +/− — Phóng to / thu nhỏ map
  • Hộp Z — Hiển thị mức zoom hiện tại
  • La bàn — Xoay map (click vào kim chỉ để reset về hướng Bắc; nút ↺↻ xoay ±5°; phím tắt [ / ])
  • Tỷ lệ (A3) — Dropdown chọn tỷ lệ in (1:200 → 1:25000) → map zoom tự động đến mức tương ứng cho khổ A3
  • Layer switcher — Đổi nền: Google Maps / OSM / Vệ tinh / Vệ tinh + tên đường

4 Thêm marker mới

1
Nhấn "+ Thêm marker" Nút chuyển đỏ, con trỏ thay đổi — ứng dụng chờ chọn vị trí.
2
Chọn vị trí • Nhấn vào bản đồ tại vị trí cần đặt
• Hoặc nhấn Vị trí hiện tại để dùng GPS
• Hoặc nhấn Chọn trên bản đồ để kéo ghim
3
Điền thông tin trong form — xem bảng các trường bên dưới.
4
Chụp / chọn ảnh (tùy chọn) Nhấn nút camera để chụp ảnh hiện trường. Ảnh upload lên GitHub.
5
Nhấn "Lưu marker" Dữ liệu ghi vào Google Sheets ngay qua Apps Script.

Các trường thông tin

  • Tên trụ — Tên định danh (bắt buộc)
  • Tủ điều khiển — Tên tủ điện quản lý trụ này
  • Cáp trụ/tủ gốc — Trụ hoặc tủ đầu chuỗi cáp. Dropdown gợi ý 3 nhóm: [tủ] (tủ điều khiển đã nhập), trụ cùng basename, trụ [cùng tủ] (khác tên nhưng cùng tủ điều khiển). Chọn xong tự tính khoảng cách Haversine.
  • Khoảng cách (m) — Tự động tính khi chọn trụ gốc; hoặc nhập tay
  • Loại trụ / Loại cần / Loại đèn — Chọn từ danh sách
  • Số lượng đèn — Số bóng đèn trên 1 trụ (1–4); ảnh hưởng đến hình dạng icon
  • Công suất — VD: 100W, 150W
  • Mã PE — Mã thiết bị PE
  • Đường / Phường Xã — Tự động tra từ tọa độ (OpenStreetMap Nominatim)
  • Ghi chú — Ghi chú bổ sung (tùy chọn)
📐
Tọa độ VN2000-X/Y tự động tính từ lat/lon khi lưu (Gauss-Krüger, múi 6°, GRS80). Không cần nhập tay.

5 Xem và chỉnh sửa marker

Nhấn vào icon bất kỳ trụ nào trên bản đồ để mở popup thông tin.

Popup hiển thị

  • Tên, ID, loại, tủ điều khiển, loại trụ/cần/đèn, số lượng đèn, công suất, mã PE
  • Đường, phường/xã; tọa độ GPS; thời gian cập nhật; người khảo sát
  • Ảnh hiện trường (nếu có)
  • Khoảng cách đến marker gốc và từng đoạn trong chuỗi

Các nút hành động trong popup

  • 🧭 Đi đến — Chỉ đường xe máy từ vị trí hiện tại (OSRM)
  • ✏️ Sửa — Mở form chỉnh sửa (cần quyền canEdit)
  • 🗑 Xóa — Xóa marker (cần quyền canDelete)
  • 🌐 Phố — Xem ảnh đường phố Mapillary tại tọa độ marker (xem mục 9)

Chỉnh sửa marker

1
Nhấn "✏️ Sửa" — Form mở với toàn bộ thông tin hiện tại.
2
Sửa trường cần thiết — Khi đổi tên trụ, app tự nhận biết và cập nhật đúng hàng trên Sheet (không tạo hàng mới).
3
Nhấn "Lưu marker" — Dữ liệu cập nhật ngay. Ảnh cũ giữ nguyên nếu không upload ảnh mới.

Di chuyển vị trí marker (kéo)

1
Mở bảng điều khiển ☰ → mục Chỉnh vị trí & Đồng bộ → nhấn "Bật chỉnh vị trí".
2
Kéo marker đến vị trí mới — badge số bên cạnh nút Cập nhật tăng lên.
3
Nhấn "Cập nhật" — Đẩy tất cả marker đã kéo lên Google Sheet, hiện tiến độ "Đang cập nhật i/N…"
4
Tắt chỉnh vị trí sau khi xong để tránh kéo nhầm.
🔒
Chỉ tài khoản admin / user / user1 mới kéo được marker. Tài khoản demo không thể kéo.

6 Tìm kiếm & Bộ lọc

Tìm kiếm nhanh

Nhập từ khóa vào ô tìm kiếm trên thanh trên, nhấn Enter hoặc nút Tìm.

  • Tên trụ — khớp một phần, không phân biệt dấu
  • ID — tìm chính xác hoặc một phần
  • Tọa độ — nhập dạng lat,lon (VD: 10.776, 106.700) để zoom đến vị trí

Bộ lọc marker

Nhấn nút Lọc ở thanh dưới để mở bộ lọc. Áp dụng bộ lọc ảnh hưởng đến:

  • Marker hiển thị trên bản đồ
  • Sơ đồ cáp (vẽ theo dữ liệu đã lọc)
  • In bản vẽ PDF (chỉ in dữ liệu đã lọc)
  • Xuất báo cáo Excel

Các tiêu chí lọc

  • Loại marker — checkbox từng loại (trụ STK, trang trí, HTLT, TTLT, tủ nổi, tủ ngầm)
  • Người khảo sát — nhập tên (khớp một phần, không phân biệt dấu)
  • Phường / Xã — nhập tên phường/xã
  • Đường — nhập tên đường
  • Tủ điều khiển — ô tìm kiếm tủ + listbox chọn nhiều (Ctrl / ⌘ + click để chọn nhiều tủ)
💡
Tìm kiếm trong bộ lọc không phân biệt dấu và chữ hoa/thường. Gõ "quan 1" cũng khớp "Quận 1".
✅
Sau khi áp dụng bộ lọc, Sơ đồ cáp và In bản vẽ tự động dùng đúng dữ liệu đã lọc — không cần chọn lại.

Hiển thị sơ đồ cáp

Nhấn ☰ → Sơ đồ cáp để bật/tắt lớp đường cáp trên bản đồ.

  • Cáp vẽ theo dữ liệu bộ lọc hiện tại (không cần chọn tủ riêng)
  • Mỗi đoạn cáp là đường nét đứt xanh đậm nối từ trụ con → trụ/tủ gốc
  • Nhãn khoảng cách (m) hiển thị tại điểm giữa đoạn cáp
🔌
Dữ liệu cáp lấy từ cột "Marker gốc" và "Khoảng cách (m)" của mỗi trụ. Trụ có Marker gốc = tên tủ/trụ cha sẽ được nối bằng đường cáp.

Sửa cáp hiện có

Khi sơ đồ cáp đang bật, nhấn "Sửa cáp" → click vào đường cáp trên bản đồ → popup hiện 2 tùy chọn:

  • 🗑 Xóa đoạn này — xóa liên kết cáp, đồng bộ GAS ngay
  • ↩ Đổi điểm gốc — click vào trụ/tủ khác để thay điểm nguồn; khoảng cách tự tính lại

Vẽ cáp mới trực tiếp trên bản đồ

Khi sơ đồ cáp đang bật, nhấn "Vẽ cáp" để vào chế độ vẽ:

1
Nhấn "Vẽ cáp" — nút đổi thành "Hủy vẽ", thanh trạng thái hướng dẫn xuất hiện.
2
Click vào trụ CON (trụ nhận cáp) — popup hiện nút 📍 Chọn làm trụ CON. Nhấn nút đó.
3
Click vào trụ/tủ GỐC (điểm nguồn cáp) — popup hiện nút 🔗 Nối cáp: [con] → [gốc]. Nhấn nút đó.
4
Cáp được tạo tự động — trường "Marker gốc" và "Khoảng cách" của trụ con được cập nhật, đồng bộ Google Sheets ngay, sơ đồ cáp vẽ lại.
5
Tiếp tục vẽ cáp khác hoặc nhấn "Hủy vẽ" / tắt sơ đồ cáp để thoát.
📏
Khoảng cách cáp được tính tự động bằng công thức Haversine từ tọa độ hai điểm — không cần nhập tay.
⚠️
Chỉ tài khoản có quyền chỉnh sửa mới thấy nút "Vẽ cáp" và "Sửa cáp".

8 In bản vẽ kỹ thuật

Nhấn ☰ → In bản vẽ để xuất PDF bản vẽ sơ đồ tuyến trạm đèn.

Dữ liệu in

Bản vẽ dùng bộ lọc hiện tại trên bản đồ — không có dropdown chọn tủ riêng. Áp dụng bộ lọc đúng trước khi mở modal in.

Các trường trong modal

  • Tỉ lệ — 1:500, 1:1000, 1:2000... Bản đồ zoom đúng theo tỉ lệ (có hỗ trợ fractional zoom). Đo bằng thước trên PDF in ra phải khớp với tỷ lệ chọn.
  • Khổ giấy — A3 hoặc A4 ngang
  • Tên bản vẽ — Hiển thị trong khung tên (title block). Nếu trống → fallback sang tên tủ đang lọc → mặc định "Sơ đồ tuyến"
  • Số bản vẽ
  • Người lập / Ngày (ngày luôn lấy hôm nay)
  • Chuyên viên phụ trách ĐB và CS khu vực
  • Tự nhớ: tất cả field (trừ Ngày) lưu localStorage — mở modal lần sau tự fill lại

Xem trước bản vẽ

Nhấn "Xem trước" — bản đồ chuyển sang chế độ in với khung viền, legend, cáp nét đứt và khung tên. Thanh nổi xuất hiện với 2 nút: Xuất PDF và Đóng xem trước.

Xuất PDF

1
Điền thông tin modal → nhấn "Xuất PDF" (hoặc từ thanh xem trước).
2
App chuyển sang bản đồ CartoDB (có CORS), zoom đúng theo tỉ lệ + khổ giấy, đợi tile load xong (event-based, ≤5s).
3
Capture toàn bộ khung bản vẽ → tạo PDF A3/A4 → tải xuống tự động.

Kiểm tra tỷ lệ in

📐
Để verify tỷ lệ: in PDF ra giấy (Print actual size — KHÔNG fit-to-page) → dùng thước đo: 1 cm trên giấy = scale × 1 cm trên thực địa.
Ví dụ 1:1000 → 1 cm trên giấy = 10 m trên thực địa. Hoặc đo giữa 2 trụ trong PDF rồi so với khoảng cách Haversine thực.

Bố cục bản vẽ

  • Tiêu đề — Tên bản vẽ + tủ điều khiển + tỉ lệ (top header)
  • Bản đồ + cáp — Vùng giữa, marker icon + đường cáp nét đứt + nhãn khoảng cách
  • Footer bottom-left (37.5%) — Bảng 3 cột: Bảng ký hiệu | Số lượng | Đơn vị. Mỗi loại marker 1 hàng (icon kích thước thật + tên + count + đơn vị "trụ"/"tủ"), hàng cuối là "Cáp nguồn" (m).
  • Khung tên (title block bottom-right 62.5%) — 4 hàng: tên tổ chức + ô ký tên trống + tên người (TT QLGT — Phó GĐ — Người lập — Bản vẽ — SHBV)
🖨
Thư viện jsPDF và html2canvas chỉ tải khi cần — không ảnh hưởng tốc độ tải app thông thường.

15 Cấu hình GPS (Phone & RTK)

App hỗ trợ 2 chế độ GPS: 📱 Phone GPS (mặc định, ±3–10m) và 🛰 RTK (Tersus Luka qua mock location, ±1–5cm khi Fixed). Chọn chế độ tại ☰ → Thiết bị GPS.

ℹ️
Mỗi chế độ có ngưỡng accuracy + precision tọa độ riêng. Marker đo bằng RTK lưu tọa độ chính xác đến mm (8 chữ số thập phân), marker phone GPS lưu cm (6 chữ số).

Quá trình lấy GPS — thanh tiến độ nổi

Khi user bấm "Thêm marker" hoặc "Vị trí hiện tại", một thanh nổi xuất hiện sát đáy màn hình hiển thị accuracy realtime:

  • ⏳ Phase tìm tín hiệu — bar màu vàng/cam, hiển thị accuracy hiện tại + mục tiêu + thời gian đã chờ. Map vẫn pan/zoom được.
  • ✓ Phase đạt mục tiêu — bar đổi xanh, countdown 3 giây auto-accept. User có thể bấm "Dùng ngay" để chấp nhận ngay, hoặc "Đợi tốt hơn" để chờ accuracy tốt hơn.
  • Auto-discard nếu user bấm "Hủy" hoặc đổi chế độ GPS giữa chừng.

Phần A — Cấu hình Phone GPS (mặc định)

Tối ưu GPS điện thoại để đạt accuracy <5m:

1
Bật chế độ High Accuracy: Settings → Location → Mode → "High accuracy" (Android) hoặc Settings → Privacy → Location Services → ON (iOS). Đảm bảo dùng GNSS thật, không chỉ Wi-Fi/Network.
2
Đứng outdoor, sky view rõ — Tránh tường, dưới cây dày, gầm cầu. Multipath làm sai số tăng 5-10× trong đô thị dày đặc.
3
Tắt Battery Saver cho browser — Battery saver throttle GPS, accuracy tệ đi đáng kể. Set Chrome / Safari "No restrictions" trong Battery settings.
4
Warm-up GPS — Trước khi khảo sát, mở Google Maps 30 giây để GPS download orbit data (A-GPS). Sau đó mở app khảo sát, fix sẽ nhanh và chính xác hơn.
5
Đứng yên 5-10s khi lấy GPS — App tự gom 8 mẫu liên tiếp, loại outlier (Median Absolute Deviation), tính trung bình. Sai số giảm ~√8 ≈ 2.83 lần so với 1 mẫu.

Phần B — Setup RTK Tersus Luka qua Mock Location (Android)

Đạt accuracy <5cm (RTK Fixed) — chỉ trên Android, cần app Nuwa của Tersus + NTRIP correction.

1
Pair Bluetooth: Bật Luka, long-press power 5s → chuyển sang chế độ pair. Settings → Bluetooth → tìm "Luka_xxx" → kết nối.
2
Cài Nuwa: Tải APK từ tersus-gnss.com (không có trên Play Store). Cài đặt và mở.
3
Connect Luka trong Nuwa: Device → Connect → chọn Luka đã pair. Hiển thị trạng thái "Connected".
4
Config NTRIP: Survey → Settings → CORS/NTRIP. Điền:
  • Server: IP NTRIP caster (VD VINAGEO: caster.vinageoit.com)
  • Port: 2101 hoặc 8080
  • Mountpoint: VD VRSNT_NRTK
  • User / Password: do nhà cung cấp cấp
5
Chờ RTK FIXED: Nuwa hiển thị quality indicator ở góc dưới — SINGLE → DGNSS → FLOAT → FIXED. Cần đạt FIXED mới có ±1-2cm. Thời gian: 30s - 2 phút tùy NTRIP và môi trường.
6
Bật Developer Options trên Android: Settings → About phone → tap "Build number" 7 lần → quay lại Settings → có mục "Developer options".
7
Set Nuwa làm Mock Location app: Settings → Developer options → "Select mock location app" → chọn Nuwa.
8
Trong Nuwa: Output to System Location: Device → Output → bật "Output to system location" (tên có thể là "Mock location output" hoặc "Share with system"). Bắt đầu phát tọa độ RTK ra Android Location Service.
9
Verify: Mở Chrome → vào https://www.maptiler.com/whereami/ hoặc app khảo sát → accuracy phải hiển thị ≈ 0.02m (2cm). Nếu vẫn 3-5m, mock location chưa hoạt động — kiểm tra bước 7-8.
10
Trong app khảo sát: ☰ → Thiết bị GPS → chọn 🛰 RTK. Từ giờ thanh tracking bar sẽ hiển thị status badge: RTK FIXED / FLOAT / DGPS.
⚠️
iOS không cho phép mock location qua app bên thứ 3 — chỉ Android. Trên iPhone vẫn dùng được Phone GPS thường.

NTRIP — chọn nguồn correction ở Việt Nam

  • VINAGEO (vinageoit.com) — phổ biến nhất với surveyor, phủ TP HCM/Hà Nội/Đà Nẵng. Subscription tháng.
  • VRSNT (Bộ TN&MT) — toàn quốc nhưng phải xin phép qua Sở TN&MT địa phương.
  • CHC NETSS / Trimble VRS Now — đi kèm hardware từ dealer.
  • Base station riêng — cần 1 GNSS receiver thứ 2 làm base, phủ bán kính ~10-15 km. Free sau invest.

Cho khảo sát ở khu vực HCMC / Bình Dương / Long An, VINAGEO là lựa chọn phù hợp nhất.

Phân biệt RTK Fixed / Float / DGPS / SPS

  • 🟢 RTK FIXED (accuracy < 5cm) — Ambiguity đã giải, độ chính xác cm. Mục tiêu khi khảo sát chính xác.
  • 🟡 RTK FLOAT (5-50cm) — Vẫn đang tìm ambiguity, chấp nhận được cho khảo sát thông thường.
  • 🔵 DGPS / SBAS (50cm-5m) — Code differential, chính xác thấp.
  • ⚪ SPS (>5m) — GPS thường, không có correction. Tương đương Phone GPS.
📡
App tự phân loại status dựa trên giá trị accuracy trả về từ browser. Không cần parse NMEA — workflow mock location ẩn hoàn toàn chi tiết hardware.

Lưu trong sheet

  • Mỗi marker lưu thêm 2 cột: Độ chính xác (m) (giá trị raw, vd 0.018) và Chế độ GPS (rtk hoặc phone).
  • Sau này có thể filter Excel để biết marker nào đo bằng RTK (vd Độ chính xác < 0.1).
  • Marker cũ chưa có 2 cột này → hiển thị rỗng, không ảnh hưởng chức năng.
